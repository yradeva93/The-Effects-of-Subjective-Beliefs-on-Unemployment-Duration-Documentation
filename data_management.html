Data Management¶

The directory src/data_management contains several files cleaning the data, creating new variables and merging to a final dataset called “final” both in ”.dta” and ”.csv” format. The structure allows for individual handling of each wave of the data as they differ in the variables used in the analysis.

Data Cleaning¶

The file “w1_qs.do” cleans the “LED_9310_s_w1_qs.dta” questionnaire part for wave 1 of the IZA Survey Dataset and creates a “w1_qs.dta” file which is prepared for merging with the other waves of the questionnaire part of the dataset. For more information please see the User Manual - Arni et al. (2013).

The file “w2_qs.do” cleans the “LED_9310_s_w2_qs.dta” questionnaire part for wave 2 of the IZA Survey Dataset and creates a “w2_qs.dta” file which is prepared for merging with the other waves of the questionnaire part of the dataset. For more information please see the User Manual - Arni et al. (2013).

The file “w3_qs.do” cleans the “LED_9310_s_w3_qs.dta” questionnaire part for wave 3 of the IZA Survey Dataset and creates a “w3_qs.dta” file which is prepared for merging with the other waves of the questionnaire part of the dataset. For more information please see the User Manual - Arni et al. (2013).

The file “wi_qs.do” cleans the “LED_9310_s_wi_qs.dta” questionnaire part for the interim wave of the IZA Survey Dataset and creates a “wi_qs.dta” file which is prepared for merging with the other waves of the questionnaire part of the dataset. For more information please see the User Manual - Arni et al. (2013).

The file “qs.do” merges the clean data from the different waves of the IZA Survey Dataset: “w1_qs.dta”, “w2_qs.dta”, “w3_qs.dta”, “wi_qs.dta”. For more information please see the User Manual - Arni et al. (2013).

The file “id_cohort.do” creates a dataset “id_cohort.dta” with the identification number, cohort, month and year of the cohort for each individual. It will be used for generating the unemployment durations.

Creating Unemployment Duration¶

The “employed_after_m.py” creates unemployment durations for the longitudinal part of the IZA Survey Dataset. It subtracts the beginning of unemployment determined by the cohort month and year (taken from “id_cohort.dta”) from the self-reported month and year of employment. It creates “employed_after_m.dta”

Final Dataset¶

The file “final.do” merges the clean data from the questionnaire “qs.dta” and unemployment durations “employed_after_m.dta” generated from the longitudinal part of the IZA Survey Dataset and creates the final cleaned and merged data “final.dta” for the purpose of the project.

Data Cleaning for Classification¶

The “classification_preprocessing.py” creates a restricted data frame, which is needed for the machine learning part in the analysis step. The four quartiles of the target variable, the observed unemployment duration, are used as categories. All variables which might have explanatory power are kept and modified if needed. Specific variables for the purpose of the classification are created.
